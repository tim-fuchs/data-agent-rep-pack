# Style Adaptation And Documentation

Before coding, check whether `CODE_STYLE/` contains files:

- If present, reuse those files as the style model for design and documentation.

For notebooks:

- Add a Markdown cell describing the purpose of each code cell block.

For classes, functions, and methods:

- Add a documentation comment.

Ask whether project-level documentation should be added to `README.md` with sections:

- What is this project about
- Prerequisites
- Install
- Start
