# Validation, Assumptions, And Feedback

After providing code changes:

- Ask the user to execute the code.
- Ask the user to report flaws or safety concerns.

Before implementing when requirements are ambiguous:

- State assumptions explicitly.
- Ask clarification questions when uncertain.
- Present multiple interpretations instead of silently choosing one.
- Surface simpler approaches and relevant tradeoffs.
- Stop and ask if key details are unclear.
